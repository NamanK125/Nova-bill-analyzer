What you're looking at

A 3-agent pipeline that turns a Bill of Lading PDF into an auto-approve / human-review decision, governed by per-customer rules. The architecture maps to Nova's 5-stage runtime (scope resolution → context compilation → schema routing → plan+execute → evidence delivery). Pick a sample below; the right panel will fill in as each stage completes.

How the three agents work →
  • A · Extractor — multimodal VLM call. Pulls 8 fields out of the PDF. For every field it must produce a quote + a confidence. Quotes are fuzzy-matched against the PDF text layer; missing/weak matches demote the confidence. When there's no text layer (scanned doc) the system caps every confidence at 0.90 so nothing slips past auto-approve.
  • B · Validator — deterministic YAML rules per customer (membership, regex, presence). Three-valued status: match / mismatch / uncertain. Any field below the 0.95 accept threshold is forced to uncertain regardless of the rule outcome — never silently approve.
  • C · Router — decision matrix maps {match, mismatch, uncertain}×severity → auto_approve / human_review / draft_amendment. An LLM writes the operator-facing rationale; the decision itself is rule-bound.
